## 已知问题与风险

### 1. `docker-compose.yml` 路径疑似错误

- 当前文件写的是：
  - `./packages/server/db/init_schema.sql:/docker-entrypoint-initdb.d/init.sql`
- 但仓库实际目录是：
  - `packages/servrer/db/init_schema.sql`
- 如果不修正，Docker 初始化数据库时可能找不到 SQL 脚本。

### 2. `servrer` 目录命名拼写异常

- 当前仓库后端目录是 `packages/servrer`，不是常见的 `packages/server`。
- 现阶段不要擅自重命名目录，否则可能破坏已有脚本、配置和路径引用。
- 但所有新文档和脚本必须注意沿用当前真实路径，避免写错。

### 3. 前端业务完成度明显低于后端骨架

- 当前 `App.vue` 仍为默认占位内容，说明前端主界面尚未进入稳定展示阶段。
- 后续如果要答辩展示，前端将是当前最明显短板之一。

### 4. AI 能力尚未达到 README 要求

- README 要求使用 `Spring AI` 生成结构化农情诊断。
- 当前成员3只完成了规则化日报生成骨架，还不是最终交付状态。
