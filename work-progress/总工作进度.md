## 已知问题与风险

### 1. 实体与数据库脚本可能不一致

- `Sensor` 实体和 `AgriMaterialSupplier` 实体都使用了 `@TableLogic` 并映射 `deleted` 字段。
- 但当前 `init_schema.sql` 中的 `sensor`、`agri_material_supplier` 表定义未看到 `deleted` 列。
- 这意味着相关 CRUD 在真实运行时可能出现字段不存在、逻辑删除失效或查询异常。

### 2. IoT 概览 SQL 存在中文状态乱码风险

- `IotDashboardMapper.selectOverview()` 中状态值出现乱码字符串。
- 如果数据库真实值是 `在线 / 离线 / 未处理`，则该统计 SQL 可能无法正确命中。
- 这会直接影响 `/iot/overview` 的在线数、离线数、未处理告警数准确性。

### 3. 前端工具文件与工程结构未对齐

- `request.js` 依赖 `@/router`，但当前仓库内未找到对应路由实现。
- 这说明前端工具层与真实页面层还没有接通。

### 4. README 与真实完成度差距较大

- README 中写到的数字孪生页面、Spring AI、完整业务管理能力，当前大部分仍未落地。
- 如果直接按 README 口径对外展示，容易造成“文档完成度高于代码完成度”的误判。

### 5. 脚本和部分文档仍是占位

- `docs/api-contract.yaml`、`docs/websocket-protocol.md`、`scripts/dev-all.js` 为空。
- `scripts/build.sh` 当前不是有效构建脚本。
- 这会影响新成员接手、统一启动和联调效率。
