## 当前目录与基础设施

### 仓库结构

- 已存在统一目录：`packages/server` 后端、`packages/web` 前端、`docs` 文档、`scripts` 脚本。
- 根目录存在 `README.md`、`Member3工作进度.md`、本文件。
- 根目录 `package.json` 当前基本为空，不能视为有可用的根级脚本编排能力。

### 后端工程基础

- 后端为 Maven 工程，入口为 `packages/server/src/main/java/com/smartfarm/SmartFarmApplication.java`。
- 已开启 `@EnableScheduling`，说明定时任务能力已启用。
- 已配置 `@MapperScan("com.smartfarm.modules.**.mapper")`。
- `application.yml` 已配置 MySQL 数据源、MyBatis-Plus、Sa-Token。
- `pom.xml` 已包含 `Spring Boot`、`MyBatis-Plus`、`Sa-Token`、`WebSocket`、`Validation`、`springdoc-openapi`。
- `pom.xml` 已使用 `jar` 打包，`mvn -DskipTests compile` 会真实编译 Java 源码。
- `pom.xml` 当前没有 `Spring AI` 相关依赖，说明 README 里要求的 AI 真接入尚未落地。

### 前端工程基础

- 前端为 Vite + Vue 3 工程，入口为 `packages/web/src/main.js`。
- 已安装依赖：`vue`、`element-plus`、`axios`、`pinia`、`vue-router`、`echarts`。

## 已知问题与风险

### 1. 实体与数据库脚本可能不一致

- 该问题已在当前代码、本机 `init_schema.sql` 和本机 MySQL 实库中同步修正。
- 如果其他环境仍在使用旧版库结构，则仍需手动补齐 `sensor.deleted` 与 `agri_material_supplier.deleted`。

### 2. IoT 概览 SQL 存在中文状态乱码风险

- 该问题已在当前代码中修正。
- 如果线上数据库仍在运行旧编译产物或未重新部署，则 `/iot/overview` 仍可能继续显示错误统计。

### 3. 前端工具文件与工程结构未对齐

- 前端基础 `router` 已补上，但路由守卫和正式登录链路仍未落地。
- 成员3 IoT 页面已经开始并已消费 `sensor_data` / `device_alert` / `sensor` / `report` 等接口。
- 当前成员3与成员5页面都存在演示化登录兜底逻辑，便于本地联调，但这本身也是一个待清理的安全风险。

### 6. 前端演示账号硬编码风险

- 成员3和成员5前端都存在本地演示账号自动登录逻辑。
- 这有利于老师现场快速演示，但如果不在最终整理前移除，会形成明显的安全与规范风险。

### 7. 前端打包体积与 warning 风险

- 当前 `npm run build` 可通过，但已有较明显的 chunk 体积 warning。
- 同时存在 `@vueuse/core` 的 Rolldown 注解 warning，虽然不阻塞构建，但说明前端打包链路还没有完全清理干净。

- README 中写到的数字孪生页面、Spring AI、完整业务管理能力，当前大部分仍未落地。
- 如果直接按 README 口径对外展示，容易造成“文档完成度高于代码完成度”的误判。

- `scripts/build.sh` 当前不是有效构建脚本。
