## 已知问题与风险

### 1. 实体与数据库脚本可能不一致

- `init_schema.sql` 中当前已包含 `sensor.deleted` 与 `agri_material_supplier.deleted`。
- 最新代码为了兼容部分旧运行库，已移除部分 IoT 查询中的 `sensor.deleted` 过滤；如果团队重新用当前脚本初始化库，两种情况都能兼容。

### 2. IoT 概览 SQL 存在中文状态乱码风险

- 最新代码已将 `/iot/overview` 中 `在线 / 离线 / 未处理` 条件改为 `_utf8mb4` 十六进制字面量，降低 Java 源码或数据库连接编码差异导致的 SQL 乱码风险。
- 如果线上仍运行旧编译产物或未重新部署，则 `/iot/overview` 仍可能继续显示错误统计。

### 3. 前端工具文件与工程结构未对齐

- 前端基础 `router`、路由守卫、登录页和主 Layout 已补上。
- 成员3 IoT 页面已经消费 `sensor_data` / `device_alert` / `sensor` / `report` 等接口。
- 成员3与成员5 API 封装仍存在演示化自动登录兜底逻辑，便于本地联调，但后续应统一收口到正式登录链路或环境开关。

### 6. 前端演示账号硬编码风险

- 成员3和成员5前端都存在本地演示账号自动登录逻辑。
- 这有利于老师现场快速演示，但如果不在最终整理前移除，会形成明显的安全与规范风险。

### 7. 前端打包体积与 warning 风险

- 同时存在 `@vueuse/core` 的 Rolldown 注解 warning，虽然不阻塞构建，但说明前端打包链路还没有完全清理干净。
